# journal-tracker v0.3.0

## Summary

`v0.3.0` focuses on reducing setup friction for non-technical users and making
it easier to expand a tracker with new journals.

This release adds:

- workbook-based settings
- workbook-driven tracked-journal selection
- OpenAlex source discovery for unmapped journals
- double-click launcher scripts for sync and discovery

## Highlights

### Workbook-based settings

You can now manage common sync defaults directly from a `Tracker Settings`
sheet in the workbook instead of relying only on JSON profiles.

Supported workbook-managed settings include:

- `Years`
- `Articles Sheet`
- `Journal Directory Sheet`
- `CSV Output`
- `Use Tracked Journals Only`
- `Crossref Mailto`

Resolution order is:

1. CLI arguments
2. JSON profile values
3. workbook settings
4. built-in defaults

### `Track?` in `Journal Directory`

The `Journal Directory` sheet now supports a `Track?` column. When workbook
settings enable tracked-journal mode, only rows marked with values such as
`yes`, `y`, `true`, or `1` are included in sync runs.

### Journal discovery workflow

New command:

```bash
journal-tracker-discover-journals --workbook data/your-tracker.xlsx
```

This command:

- scans `Journal Directory`
- finds journals missing from `config/openalex_sources.json`
- searches OpenAlex Sources for likely matches
- writes a `Journal Match Suggestions` sheet into the workbook

The suggestions sheet includes a top suggested source, alternative candidates,
publisher information, and a short confidence note.

### Double-click launchers

For easier local use, the project now includes launcher scripts for the default
local workbook:

- macOS
  - `scripts/run_sync.command`
  - `scripts/discover_journals.command`
- Windows
  - `scripts/run_sync.bat`
  - `scripts/discover_journals.bat`

## Validation

- `ruff check`
- `pytest` (`19 passed`)
- sample workbook updated to include `Tracker Settings` and `Track?`

## Notes

- This release keeps existing JSON profile support.
- Existing workbooks remain usable; workbook settings are an additional layer,
  not a breaking replacement.
- Journal discovery writes suggestion output into the workbook but does not yet
  write approved mappings back to config automatically.
